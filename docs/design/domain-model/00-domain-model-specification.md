# Domain Model Specification (Main)

## 1. Introduction
This document establishes the foundations of the business domain. It contains the shared language (Ubiquitous Language) and the global rules that apply across all sub-domains and aggregates.

---

## 2. Core Glossary & Shared Value Objects
These objects are immutable and used throughout the entire system to ensure consistency.

| Value Object   | Base Type | Responsibility / Constraints                                                |
|:---------------|:----------|:----------------------------------------------------------------------------|
| **InternalId** | String    | System-generated unique identifier. Must be used for all entity references. |
| **Money**      | Object    | Holds `amount` (BigDecimal) and `currency`. Prevents invalid arithmetic.    |
| **DateRange**  | Object    | Ensures `startDate` is always before `endDate`.                             |

Rules that require coordination between multiple aggregates or external infrastructure.

* **Goal:** Governs how users are identified and verified.

## 5. Standard Invariants (Global Rules)
1. **No ID Mapping:** Internal IDs generated by the system must not be modified or mapped to external database IDs within the domain.
2. **Persistence Ignorance:** The domain must not know about SQL, JSON, or any specific storage technology.
3. **Reference by Identity:** Aggregates can only reference other aggregates by their `InternalId`.
